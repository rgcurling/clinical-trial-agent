### What the system does

TrialMatch AI takes a free-text patient clinical note and returns a ranked list of relevant recruiting clinical trials with plain-English eligibility explanations written at an 8th-grade reading level.

**Pipeline:**
1. Patient note → **Claude Haiku** extracts structured profile (age, conditions, biomarkers, stage, medications)
2. ClinicalTrials.gov API fetches up to 50 live recruiting trials; BiomedBERT (PubMedBERT sentence embeddings) re-ranks them by semantic similarity to the patient profile *(TREC/M3 eval mode: BiomedBERT retrieves directly from the 26,148-trial static corpus)*
3. **Claude Sonnet** (Agent 1) evaluates inclusion/exclusion criteria per trial → eligibility score 0–1 + clarifying questions (combined in one call)
4. GPT-4o (Agent 2) independently reviews each assessment → accepts, overrides, or flags uncertain
5. Resolver merges both agents; hard-exclusion filter removes ineligible trials
6. Explainer generates patient-facing cards (FK grade ≤ 8 target); **Haiku** rewrites if readability target is missed

### M3 Ablation Study — TREC Clinical Trials 2021 (Topics 26–40)

4 experimental configurations evaluated on 15 topics from the TREC 2021 benchmark.

| Configuration | Retriever | Critic | P@5 | NDCG@5 | Est. Cost |
|--------------|-----------|--------|-----|--------|-----------|
| Baseline | TF-IDF | None | 0.571 | 0.460 | $2.19 |
| R1 — BiomedBERT | BiomedBERT | None | **0.667** | **0.613** | $2.49 |
| R2 — Critic | TF-IDF | GPT-4o | 0.523 | 0.423 | $3.25 |
| R3 — Combined | BiomedBERT | GPT-4o | 0.643 | 0.582 | $3.58 |

**Key findings:**
- BiomedBERT alone over Baseline: **+0.096 P@5 (+16.8%), +0.153 NDCG@5 (+33.3%)** — semantic retrieval is the biggest driver
- Adding the GPT-4o critic to TF-IDF (R2) slightly *hurts* performance vs Baseline: -0.048 P@5 — critic conservatism penalises recall when retrieval quality is low
- Combined (R3) outperforms Baseline but falls short of BiomedBERT alone — critic adds value only when retrieval is already strong
- **Takeaway: retrieval quality is the bottleneck, not matching sophistication**

### Confusion Matrix (Matcher vs. TREC Physician Judgments)

Evaluated on top-5 retrieved trials across all topics using R1 results.

- **False Positives: 6** — matcher says eligible, physician says not relevant → wasted screening visit risk
- **False Negatives: 11** — matcher says excluded, physician says relevant → missed trial opportunity
- FN > FP: system is **conservative** (safe bias for clinical use — better to miss a trial than send an ineligible patient)

### Explanation Quality Metrics

Evaluated on 67 patient-facing explanations generated by the Combined (R3) run.

| Metric | Result | Target | Pass Rate |
|--------|--------|--------|-----------|
| Flesch-Kincaid Grade Level | 8.37 | ≤ 8.0 | 62.7% |
| BERTScore F1 (faithfulness) | 0.799 | ≥ 0.85 | 1.5% |

**Interpretation:**
- Readability is close — FK 8.37 vs target 8.0. Most explanations are near the target; a tighter prompt constraint on sentence length would push this over.
- BERTScore faithfulness is low (0.799). This reflects that patient-facing explanations deliberately simplify clinical language, so word-level overlap with the source eligibility text is inherently low. BERTScore is arguably the wrong metric for paraphrase-based simplification.

### Production System (post-M3)

- **FastAPI REST service** — `POST /match` accepts patient text, returns ranked trials + explanations
- **Containerized** — multi-stage Docker build, <1GB image, <15s cold start
- **Google Cloud Run** — serverless, scales to zero, ~$0/month at low traffic
- **Live data** — ClinicalTrials.gov API v2 fetches live recruiting trials; BiomedBERT then re-ranks them by semantic similarity before Claude eligibility assessment (not the static TREC corpus)
- **Two-tier model design** — Claude Haiku for extraction, criteria parsing, and readability simplification; Claude Sonnet reserved for per-trial eligibility matching (~75% cost reduction vs. single-model approach, ~$0.06–0.08/patient)
- **Clarifying questions included at no extra cost** — folded into the Sonnet match call, eliminating a separate API round-trip per trial
- **32 automated tests** — 19 pipeline unit tests + 13 API integration tests, all passing
